# Projectmaster Dependencies

`projectmaster` is a routing and orchestration skill. It assumes the following skills are also installed on the machine.

## Required Core Skills

- `brainstorming`
- `writing-plans`
- `subagent-driven-development`
- `executing-plans`
- `requesting-code-review`
- `verification-before-completion`

## Required Engineering Skills

- `grill-with-docs`
- `setup-matt-pocock-skills`
- `tdd`
- `diagnosing-bugs`
- `handoff`

## Recommended Depending on Path

- `improve-codebase-architecture`
- `systematic-debugging`

## Why They Are Separate

`projectmaster` does not duplicate the content of these skills. It tells the agent when to use them and how to sequence them. If you sync `projectmaster` to another device, sync these dependencies too or the workflow will be incomplete.
